# LeetCode Patterns

## Two Pointers
- Common on sorted arrays or linked lists.
- Typical moves: shrink window from both ends.
- Watch for duplicates and overflow.

## Sliding Window
- Fixed vs variable window.
- Maintain counts or running sum.
- Track best answer while expanding; shrink when invalid.

## Hash Map / Set
- For complements, frequency counting, and uniqueness.
- Beware of default values and missing keys.

## Stack
- Monotonic stack for next greater/smaller.
- Parse parentheses, remove invalids.

## BFS / DFS
- Use BFS for shortest paths in unweighted graphs.
- Use DFS for enumeration or cycle detection.

## Binary Search
- Search on index or answer space.
- Invariants: low/high boundary conditions.

## Dynamic Programming
- Define state, transition, and base cases.
- Top-down with memo or bottom-up table.

## Greedy
- Prove local choice is optimal.
- Often combined with sorting.

## Backtracking
- Choose, explore, un-choose.
- Use pruning to cut branches.

## Bit Manipulation
- XOR tricks, bitmask DP.
- Watch signed vs unsigned.
